# 2025 코딩 보조 도구 평가 기록 (Obsidian용)
* 작성일: 2025-12-27
* 목적: 2025년 한 해 동안 사용/검토한 AI 코딩 도구들의 **경험 기반 평가**를 정리하고, 향후 선택지를 **Copilot 중심으로 수렴**시키기 위한 기준 문서화
---
## 0) 최종 의사결정 요약
* **주력 후보:** GitHub Copilot (사실상 메인 고정)
* **전략 전환:** 부족하다고 느꼈던 에이전틱 기능은 **내가 직접 수행(워크트리/체크리스트/스크립트)**하는 방식으로 전환
* **서브 에이전트 연구용으로만 유지:** Factory.ai Droid, Claude Code
* **그 외 에이전틱 IDE/도구:** 추가 고려하지 않는 방향으로 정리
---
## 1) 평가 기준(내가 중요하게 보는 축)
1. **완주율:** 작업이 끊기지 않고 끝까지 수행되는가
2. **환경 적합성:** 특히 Windows/WSL/터미널/빌드 환경에서 문제 없는가
3. **비용/사용량 예측 가능성:** 크레딧/쿼터/과금 때문에 작업이 중단되지 않는가
4. **DX(개발 경험):** 사용 흐름이 매끄럽고 신뢰할 수 있는가
5. **후회 없음:** 선택 후 “괜히 썼다”는 감정이 남지 않는가
---
## 2) 도구별 평가 로그
### GitHub Copilot
* **총평:** 2025년 1년 동안 사용해본 도구 중 **유일하게 “절대 실패 안 하는” 도구**
* **근거**
  * 기능 동작 실패가 없었음
  * 선택 시 후회가 없었음(안전한 기본값)
* **결론/액션**
  * 주력 도구로 수렴
  * (필요 시) 긴급 재구독/추가를 최우선으로 검토
---
### Windsurf
* **장점**
  * **Codemap 기능은 확실히 훌륭**(사람이 레거시를 정리/이해하는 데 도움)
* **단점/이슈**
  * **WSL 내에서 에이전트가 잘 동작하지 않는 문제**
  * **SWE-1.5가 작업 도중에 죽는 문제**가 잦음
  * Copilot 대비 에이전트 성능이 떨어진 체감
  * **2025년 초부터 지금까지** 기능적 부족이 깔끔하게 해결되지 않았다고 평가
  * 결과적으로 **DX 개선 실패**로 판단
* **결론/액션**
  * 향후 사용/구독은 더 이상 고려하지 않는 방향
  * 이미 결제된 비용이 있다면, 레거시 정리에서 Codemap 기반 산출물만 뽑고 종료
---
### Cursor (2025-07 이후 미사용)
* **장점/관심 요소(과거)**
  * 에이전틱 코딩을 주도/정의하는 기업 이미지
  * 여러 LLM을 돌려 하나를 선택하는 기능 등 “진화 속도” 인상
* **단점/이슈**
  * **BM(비즈니스/운영) 측면 신뢰가 꺾인 문제**
* **결론/액션**
  * 7월 이후 사용 중단
  * 에이전틱 IDE 도구 추가 탐색을 줄이기로 한 결정에 따라, 재검토 우선순위 낮음
---
### Claude (Pro / Claude Code)
#### Claude (대화형)
* **장점**
  * 대화형 아이디어 논의/정리에는 활용 가능
* **단점**
  * 컨텍스트 효율이 낮게 느껴짐
  * 코딩 목적에선 기대 대비 효율이 떨어진다고 체감
* **결론**
  * 코딩 메인보단 “대화형 보조”에 가까움
#### Claude Code
* **단점(결정적)**
  * **사용량 문제로 일이 끊기는 경험이 잦았던 기억**
* **결론/액션**
  * 메인 도구로는 부적합
  * 다만 **서브 에이전트 기능 연구용**으로만 리스팅 유지
---
### Factory.ai Droid
* **인상**
  * 내가 원하는 **멀티 에이전트 코딩 시스템과 유사한 기능 셋**이 존재하는 것으로 보임
* **우려**
  * **비용 우려로 확신이 안 섬**
* **결론/액션**
  * 메인 채택이 아니라 **서브 에이전트 연구 목적**으로만 리스팅 유지
---
### JetBrains AI (Junie)
* **우려**
  * **사용량 이슈**로 선택이 망설여짐
* **결론/액션**
  * 보류(메인 후보로 확정하기엔 불확실성)
---
### Zed
* **단점**
  * **DX가 매끄럽지 못했음**
* **결론/액션**
  * 우선순위에서 제외
---
### Aider
* **상태**
  * 잘 모르겠음(충분한 평가 경험 없음)
* **결론/액션**
  * 당장 추가 탐색 필요성 낮음(수렴 전략 우선)
---
### Ampcode / OpenCode / Goose / Warp
* **상태**
  * 제대로 써본 적 없음(경험 부족)
* **결론/액션**
  * 에이전틱 도구 탐색을 확대하기보다는 Copilot 중심 수렴 전략을 우선
---
## 3) 추가 도구(요청 반영)
### Amazon Q
* **평가:** 잘 모르겠음(경험/검증 부족)
* **결론/액션**
  * 당장 우선순위 낮음
### Gemini CLI
* **장점**
  * **무료인 건 좋음**
* **단점**
  * **성능이 별로 좋지 못했음**
* **결론/액션**
  * 메인/서브 모두에서 우선순위 낮음(무료이지만 시간 대비 효율이 떨어짐)
### Google Antigravity
* **평가**
  * **어떻게 활용하는지 감을 못 잡음**
* **결론/액션**
  * 당장 우선순위 낮음(명확한 사용 시나리오 부재)
### OpenAI Codex
* **평가**
  * **생각보다 많이 좋아졌음**
  * **ChatGPT 구독자라면 메인으로 쓰는 것을 고려할 만함**
* **결론/액션**
  * Copilot을 기준선으로 두되,
  * “대량 패치/반복 변경/CLI 기반 작업”에서 보조 또는 일부 구간 메인 후보로 고려 가능
---
## 4) 향후 운영 계획(간단)
* **메인:** GitHub Copilot
* **에이전틱 역할:** 내가 직접 수행(Worktree 기반 병렬화 + PR 단위 작업 + 체크리스트 검증)
* **서브 에이전트 연구:** Droid, Claude Code
* **추가 고려 최소화:** 그 외 에이전틱 IDE/도구는 더 이상 확장하지 않음
* **예외(필요 시):** OpenAI Codex는 “많이 좋아졌음” 평가에 따라 작업 유형에 한해 재도입 가능
