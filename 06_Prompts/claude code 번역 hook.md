
claude-code의 효율성을 위해 번역 hook을 개발하고 싶어. 기대하는 작업 과정은 다음과 같아. 
1. 유저가 자신의 모국어로 작성 
2. hook이 번역 api를 통해 프롬프트 번역 
3. 번역된 프롬프트를 보여주고 이에 대해서 승인 요청 
4. 이를 사용하여 작업 수행 이후 작업은 번역없이 계속 수행 아래의 과정은 모든 작업을 마친 후 마지막 단계에 실행 
5. 모든 작업 완료 후, 최종 output 번역 
6. 번역된 결과물 출력 
만약 유저가 요청하는 경우, 번역 이전 output 보여줌.

위 아이디어 구현에 hooks가 적합한지 검토하고, 다른 방법들 (agent나 tools, mcp 등)을 고려해서 가장 적합한 수단이 무엇인지 추천해줘. 이때 하나만 고를 필요는 없어.
