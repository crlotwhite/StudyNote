## "자동화된 로그" (Zero Friction)
가장 낮은 단계이자, 가장 중요한 '원시 데이터' 확보 단계
"뇌를 거치지 않고 일단 텍스트를 덤프(Dump)하는 단계"입니다. 
정리하려는 강박을 버리는 것이 **핵심**입니다.

방법: AI 회고록 (Conversation Dump)
하루 200개의 대화가 끝난 후, 마지막에 프롬프트 한 줄만 치세요.
> "오늘 우리가 나눈 대화 중, 내가 몰랐던 기술적 사실 3가지와 내가 작성한 핵심 코드 스니펫만 마크다운으로 요약해줘."

- **Obsidian 설정:**
    - **Core Plugin:** 'Daily Notes(일일 노트)'를 켜세요.
    - **단축키:** `Ctrl/Cmd + P` → `Open today's daily note` (자주 쓰므로 단축키 지정 추천, 예: `Alt + D`)
- **실천 가이드:**
    1. 하루 종일 ChatGPT와 수백 턴의 대화를 나눕니다.
    2. 대화가 끝날 때쯤 AI에게 요청합니다: _"오늘 대화 내용 중 VITS 구현에 필요한 핵심 개념 3줄 요약해줘."_
    3. `Alt + D`를 눌러 오늘 날짜 노트(`2025-11-24`)를 엽니다.
    4. **`# Log`** 헤더 밑에 그냥 붙여넣습니다. (서식, 줄바꿈 신경 쓰지 마세요.)
- **시스템 역할:** `obsidian-git`이 백그라운드에서 이 덤프 파일도 자동으로 커밋합니다. "내가 오늘 뭔가 하긴 했다"는 기록이 자동으로 남습니다.

## "TIL (Today I Learned) 리포지토리" (Low Friction)
개발자에게 가장 권장되는 방식입니다. 
블로그는 부담스럽지만, GitHub 잔디 심기는 동기 부여가 됩니다.
"작은 문제 해결의 순간을 캡처하는 단계"입니다. 
여기가 녹색 잔디(Green Squares)가 심어지는 주 무대입니다.

방법: Study-Log 리포지토리 운영
1. GitHub에 TIL 또는 Study-Log라는 리포지토리를 하나 만듭니다.
2. 폴더 구조를 주제별로 잡습니다: /DSP, /VITS, /Agent

형식: 거창한 글이 아니라, "문제-해결" 구조의 숏 노트입니다.
예시:
```
[250520] STFT 윈도우 크기 문제
문제: 윈도우가 너무 크면 시간 해상도가 뭉개짐.
해결: 윈도우 길이를 20ms로 줄이고 Hop size를 5ms로 설정하니 파열음이 잘 보임.
참고: (오늘 ChatGPT와 나눈 대화 링크)
```

- **Obsidian 설정:**
    - **폴더:** `10_Concepts` (개념), `30_Experiments` (실험)
    - **Plugin:** `Execute Code` 활용.
- **실천 가이드:**
    1. 사인파를 그리다가 `Click Noise`가 발생했습니다.
    2. `10_Concepts` 폴더에 `Click_Noise.md` 파일을 만듭니다. (파일명이 곧 주제)
    3. 내용은 딱 3부분만 적습니다.
        - **문제:** 파형을 그냥 자르니 틱 소리가 남.
        - **원인:** 진폭이 0이 아닌 지점에서 끊겨서 급격한 변화 발생.
        - **해결(Code):** `Execute Code` 블록을 열고 `Windowing` 적용한 코드 5줄 작성 후 실행(Run).
    4. **끝.** 닫으면 `obsidian-git`이 알아서 Push 합니다.
- **시스템 역할:** 별도의 TIL 리포지토리를 관리할 필요 없이, 이 폴더 자체가 고품질의 TIL이 됩니다.
## "오픈 소스형 문서화" (High Impact)
공부한 내용을 남에게 설명하듯 작성하는 것입니다.
"남에게 보여줄 수 있는 수준의 정제된 결과물"입니다.

방법: Readme Driven Learning

1. 코드를 짜기 전에 README.md부터 작성합니다.
2. "나는 오늘 Python으로 PSOLA를 구현할 것이다."라고 선언하고, 구현해야 할 함수 목록을 적습니다.
3. 공부가 끝난 후에는 그 함수의 작동 원리를 주석(Docstring)으로 답니다.

> 핵심: 코드가 곧 공부 흔적입니다. 주석을 아주 상세하게(논문 내용을 인용해서) 다는 것이 요령입니다.

- **Obsidian 설정:**
    - **파일:** 루트 경로의 `README.md`
    - **Plugin:** `Obsidian Gist`
- **실천 가이드:**
    1. 어느 정도 개념(TIL)이 쌓여서 **"PSOLA 구현체"**가 완성되었습니다.
    2. 코드가 길어서 노트에 다 넣기 지저분합니다. GitHub Gist에 접속해 `psola.py`를 올립니다.
    3. `README.md`를 엽니다.
    4. **목차**를 만들고, 지금까지 쓴 TIL 노트들을 링크(`[[Click_Noise]]`, `[[OLA_Algorithm]]`)로 연결합니다.
    5. 핵심 구현 코드는 `gist:Noel/abcd123` 형식으로 붙여넣어 임베딩합니다.
- **시스템 역할:** 방문자는 `README.md`를 통해 프로젝트의 구조를 보고, Gist로 깔끔한 코드를 보며, 링크를 타고 상세한 삽질기(TIL)까지 볼 수 있습니다. 가장 이상적인 포트폴리오 구조입니다.

## 최종 Workflow

이 시스템이 돌아가는 하루 루틴은 다음과 같습니다.

1. **[Zero]** AI와 대화하며 나온 파편적인 지식 → **Daily Note**에 복붙. (자동 커밋)
2. **[Low]** 파이썬으로 파형 만지다 해결된 문제 → **`10_Concepts/주제.md`**에 코드 실행 결과와 함께 기록. (자동 커밋 = 잔디 심기)
3. **[High]** 주말에 1시간, `README.md`를 열고 이번 주에 만든 개념 노트들을 순서대로 나열하여 목차 정리. (프로젝트화)
